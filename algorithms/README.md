# 算法学习指南

本目录包含了常见的算法实现示例，使用 Go 语言编写。

## 目录结构

```
algorithms/
├── sorting/                # 排序算法
├── search/                 # 查找算法
├── dynamic_programming/    # 动态规划
├── greedy/                 # 贪心算法
└── graph/                  # 图算法
```

## 算法分类

### 1. 排序算法 (Sorting)

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 | 说明 |
|------|-----------|-----------|--------|------|
| 冒泡排序 | O(n²) | O(1) | 稳定 | 简单但效率低 |
| 选择排序 | O(n²) | O(1) | 不稳定 | 每次选择最小元素 |
| 插入排序 | O(n²) | O(1) | 稳定 | 适合小规模数据 |
| 快速排序 | O(n log n) | O(log n) | 不稳定 | 平均性能最好 |
| 归并排序 | O(n log n) | O(n) | 稳定 | 稳定且性能好 |

**运行示例**:
```bash
cd algorithms/sorting
go run main.go
```

### 2. 查找算法 (Search)

| 算法 | 时间复杂度 | 空间复杂度 | 要求 |
|------|-----------|-----------|------|
| 线性查找 | O(n) | O(1) | 无要求 |
| 二分查找 | O(log n) | O(1) | 数组必须有序 |

**运行示例**:
```bash
cd algorithms/search
go run main.go
```

### 3. 动态规划 (Dynamic Programming)

动态规划是解决最优化问题的一种方法，通过将问题分解为子问题并存储子问题的解来避免重复计算。

**经典问题**:
- 斐波那契数列
- 爬楼梯问题
- 最长公共子序列 (LCS)
- 0-1背包问题
- 最长递增子序列
- 编辑距离

**运行示例**:
```bash
cd algorithms/dynamic_programming
go run main.go
```

### 4. 贪心算法 (Greedy)

贪心算法在每一步都做出当前看起来最好的选择，希望这样能得到全局最优解。

**经典问题**:
- 活动选择问题
- 找零问题
- 分数背包问题
- 最小生成树 (Prim, Kruskal)
- 最短路径 (Dijkstra)

**运行示例**:
```bash
cd algorithms/greedy
go run main.go
```

### 5. 图算法 (Graph)

图算法用于解决图结构相关的问题。

**经典算法**:
- 深度优先搜索 (DFS)
- 广度优先搜索 (BFS)
- 拓扑排序
- 最短路径 (Dijkstra, Floyd-Warshall)
- 最小生成树 (Prim, Kruskal)

**运行示例**:
```bash
cd algorithms/graph
go run main.go
```

## 学习路径建议

### 第一阶段：基础算法
1. **排序算法** - 理解基本排序思想
2. **查找算法** - 掌握线性查找和二分查找

### 第二阶段：进阶算法
3. **动态规划** - 理解状态转移和子问题
4. **贪心算法** - 理解局部最优和全局最优

### 第三阶段：高级算法
5. **图算法** - 理解图的遍历和最短路径

## 复杂度分析

### 时间复杂度
- O(1): 常数时间
- O(log n): 对数时间
- O(n): 线性时间
- O(n log n): 线性对数时间
- O(n²): 平方时间
- O(2ⁿ): 指数时间

### 空间复杂度
- O(1): 常数空间
- O(n): 线性空间
- O(n²): 平方空间

## 算法优化技巧

1. **空间换时间**: 使用额外的空间来减少时间复杂度
2. **时间换空间**: 使用更多的时间来减少空间复杂度
3. **预处理**: 提前计算和存储结果
4. **剪枝**: 提前终止不必要的计算
5. **记忆化**: 存储已计算的结果避免重复计算

## 练习建议

1. **理解原理**: 先理解算法的核心思想
2. **手写代码**: 不看示例，自己实现
3. **分析复杂度**: 分析时间和空间复杂度
4. **优化改进**: 思考如何优化算法
5. **实际应用**: 找到实际应用场景
